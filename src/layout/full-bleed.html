Full bleed

The problem

How to create a single fixed-width column in the center of the page? You've seen this layout everywhere: online magazines, documentation, news sites, and blogs. You're looking at it right now, on this site!

There's a complicating factor, howeverâ€”not all content should be constrained. We should allow images, videos, and/or custom widgets to break free and fill the available width:

For the next examples we'll use the following markup (note, we want the <img /> to span/bleed): 

<body>
  <header>...</header>
  <main>
    <h1>...</h1>
    <p>...</p>
    <img />
    <p>...</p>
  <main>
  <footer>...</footer>
</body>

The principle of least power

main > * {
  margin-left: auto;
  margin-right: auto;
  max-width: 65ch;
}

main > img {
  max-width: 100ch;
}
Pros
  • Lightweight
  • Easily understandable
  • Good browser support
Cons
  • Content padding on small screen sizes (solvable with @media)

The grid

The center column is the actual content area, while the two outer columns act as padding, but also create the effect of margin: 0 auto. 

main {
  display: grid;

  /* Padding the whole content */
  grid-template-columns:
    1fr
    minmax(calc(100% - 4rem), 65ch)
    1fr;
  grid-column-gap: 2rem;

  /* OR not padding bleed elements */
  grid-template-columns:
    minmax(2rem, 1fr)
    minmax(auto, 65ch)
    minmax(2rem, 1fr);

  & > * {
    grid-column: 2;
  }

  & > img {
    grid-column: 1 / -1; /* Whole screen*/
    justify-self: center;
    width: 100%;
    max-width: 100ch;
  }
}

The paddings on the side should differ on various screen sizes. On smaller screens, we want to limit the wasted space, while on bigger screens more padding can improve the visual quality. A solution could be combining vmin/vmax or vw units who depend on the size of the viewport: 

main {
  ...
  grid-template-columns:
    minmax(min(2rem, 2vw), 1fr)
    minmax(auto, 65ch)
    minmax(min(2rem, 2vw), 1fr);
  ...
}

Now we have a dynamic and maintainable layout!
