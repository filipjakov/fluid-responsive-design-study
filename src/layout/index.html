Layout/Component patterns

(Sometimes) It's better to provide suggestions rather than diktats about the way the visual design is laid out. An overuse of @media breakpoints can easy come about when we try to fix designs to different contexts and devices. By only suggesting to the browser how it should arrange our layout boxes, we move from creating multiple layouts to single quantum layouts existing simultaneously in different states.

Patterns:

  • The (Un)Holy Albatros AKA container query for the poor
  • Full bleed
  • Responsive multi-column grid system or RAM (repeat, auto, minmax)
  • The Raven Technique
