Fluid font size

The core trickery comes from viewport units. You can literally set type in viewport units (e.g. font-size: 4vw), but the fluctuations in size are so extreme that itâ€™s usually undesirable.

How we currently do it:

:root {
  font-size: 100%;
}

h1 {
  font-size: 2.25rem; // 36px
}

@media (min-width: 700px){
  h1 {
    font-size: 3.5rem; // 56px
  }
}

@media (min-width: 1000px){
  h1 {
    font-size: 4.75rem; // 76px
  }
}

1. Rise until lock

font-size:

:root {
  font-size: calc(100% + .5vw);
}

@media (min-width: 1200px) {
  :root {
    font-size: 1.5rem;
  }
}

2. CSS Locks

Below is a CSS lock that interpolates between a font-size of 16px and 24px between the two screen sizes of 320px and 1200px. The locking is handled by the media query directly below it, without it the growth would continue at the same rate forever.

font-size:

:root {
  font-size: calc(
    16px +
    (24 - 16) * (100vw - 320px)/(1200 - 320)
  );
}

@media (min-width: 1200px) {
  :root {
    font-size: 24px;
  }
}
