CSS Length Units

A CSS unit determines the size of a property you’re setting for an element or its content. For example, if you wanted to set the property margin of a paragraph, you would give it a specific value. This value includes the CSS unit.

Absolute vs. Relative Units

When considering all the options for which units to use, it’s important to consider the two categories of units: absolute and relative.

Absolute units:

Units that are “absolute” are the same size regardless of the parent element or window size. This means a property set with a value that has an absolute unit will be that size when looked at on a phone or on a large monitor (and everything in between!). Some of the available units are:

  • px - Pixel
  • pt - Point
  • pc - Pica
  • cm - Centimeter
  • mm - Millimeter
  • in - Inch

Relative units:

Motivation:
  • setting type in apsolute units (px) prevents browser settings from making font size adjustments
  • setting type in relative units maintains greater design fidelity as users use browser zoom
  • absolute units can be less favourable for responsive sites because they don’t scale when the screen size changes

1. Typography related:

  • em and rem
  • ex
  • ch

2. Viewport-size related:

  • vh & vw
  • vmin & vmax

Conclusion:

Overall, when and how you choose your CSS units will come down to a couple questions:

  • Do I want what I’m styling to scale when the viewport size changes?
  • If I do want it to scale, what do I want it to scale relative to in the app:
    • the viewport?
    • the content (text)?
    • the parent or anestor element?
    • or combinations of the items listed above?

Once you’ve answered these questions, it’s a lot easier to nail down which unit to use.
