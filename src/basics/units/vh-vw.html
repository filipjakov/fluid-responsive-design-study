vh - viewport height

Relative to the height of the viewport (window or app size). 1vh = 1/100 of the viewport’s height

So if the height of the viewport is 1080px, 10vh represents 108px, or 10% of the viewport height.

A common pattern:

Imagine a page with the following markup:

<body>
  <header>...</header>
  <main>...</main>
  <footer>...</footer>
</body>

There may be a case in <main> may not fill our viewport because there is not enough content in it to do so ("sticky footer") A common pattern is to do the following:

body {
  display: flex;
  flex-direction: column;
  min-height: 100vh;
}

main {
  flex-grow: 1;
}

which makes <main> fill the the viewport even though it's content does not dictate so.

vw - viewport width

Relative to the width of the viewport (window or app size). 1vw = 1/100 of the viewport’s width

So if the width of the viewport is 1440px, 10vw represents 144px, or 10% of the viewport width.

Example:

  • Changing the font-size depending on the window width (fluid typography): 
    :root {
  font-size: calc(1rem + .5vw) /* 1440*0.05=7,2 */
}
  • Useful to constrain the height-to-width ratio of an element: 
    .full-width {
  width: 100vw;
  height: calc(100vw * (9/16));
}
