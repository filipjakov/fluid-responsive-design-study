rem vs. em – The Great Debate

One of the best practices to typography on the web is to use relative units like rem and em.

What is em?

An em is a unit of typography, equal to the currently specified point-size.

What it means is: 1em = 20px if a selector has a font-size of 20px.

Consider the following:

h1 { font-size: 2em } /* ??? */

We have to look at the parent element in order to compute the h1's font-size. Let’s say the parent element is <html>, and its font-size is set to 16px. 

html { font-size: 100% } /* 16px by default */
h1 { font-size: 2em } /* 16px * 2 = 32px */
EM values inherit from their parent

em can also be used to specify values for other properties in addition to font-size. margin, padding and even line-height are such properties.

For this reason, the primary purpose of em units should be to allow for scalability within the context of a specific design element.

Example: button with px vs em

padding: 8px 16px;
/* Simulates margin */
box-shadow: 0 0 0 16px greenyellow;

& svg {
  height: 16px;
  margin-left: 16px;
}
padding: .5em 1em;
/* Simulates margin */
box-shadow: 0 0 0 1em greenyellow;

& svg {
  height: 1em;
  margin-left: 1em;
}
NOTE: this is just a showcase, pls never create such components. The problem of inheritance with font-size could get out of hand if applied along multiple levels of ancestory. Em "could" make more sense in "top-level" components such as header/footers, who basically have no ancestors.

What is rem?

rem means Root EM. It’s built to provide some relief to the em inheritance problem that many faced.

It is a unit of typography equal to the root font-size. This means 1rem is always equal to the font-size defined in <html>. Think of it as ems, but better. Actually, just forget about ems!

Consider the same code as above, written in rems instead. What are the computed margin-bottom values now?

h1 {
  font-size: 2rem;
  margin-bottom: 1rem; /* 1rem = 16px */
}
Depends on a single source -> easily understandable and scalable

Conclusion

  • Don't use ems
  • Use rems for almost everything
