em - relative to the font size of its direct or nearest parent

DISCLAIMER: don't do this, a better approach would be to use rems

While the value remains 0.773em, the actual font size is calculated at 77.3% of its direct parent, which in turn scales from its parent. This continues up the DOM tree whenever a parent has a defined font-size.

PX values do not inherit from their parent

DISCLAIMER: don't do this, a better approach would be to use rems

EM scales by updating one value

html { font-size: 100%; } /*16px*/
h1 { font-size: 2.074em; }
small { font-size: 0.833em; }
.box { padding: 1.25em; }
@media screen and (min-width: 1400px) {
  html { font-size: 125%; } /*20px*/
}

Ralculating every PX value

html { font-size: 16px; }
h1 { font-size: 33px; }
small { font-size: 13px; }
.box { padding: 20px; }
@media screen and (min-width: 1400px) {
  html { font-size: 20px; }
  h1 { font-size: 41px; }
  small { font-size: 17px; }
  .box { padding: 25px; }
}

Navigation example

I’ve never been a fan of ems, especially when it comes to font-sizes. As seen in the examples above, one quickly loses track of what size is what, and the whole thing becomes unmanageable.

The key value em units offer is they allow sizing values to be determined by a font size other than that of the html element. For this reason, the primary purpose of em units should be to allow for scalability within the context of a specific design element. For example, you might set the padding, margin and line-height around a navigation menu item to use em values.
